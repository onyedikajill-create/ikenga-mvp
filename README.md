# IKENGA MVP

IKENGA MVP is a production-ready Next.js 16 landing and intake surface for the IKENGA AI launch. It includes:

- A polished public homepage
- A live waitlist capture flow backed by Supabase
- Protected internal engine/admin API routes
- A protected IKENGA content-generation API powered by Anthropic
- Health, sitemap, robots, and social metadata endpoints for deployment

## Run locally

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Required environment variables

Create `.env.local` from `.env.example` and set:

```bash
NEXT_PUBLIC_SITE_URL=https://your-domain.com
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
IKENGA_ADMIN_API_TOKEN=replace-with-a-long-random-token
ANTHROPIC_API_KEY=your_key_here
```

Optional:

```bash
IKENGA_ANTHROPIC_MODEL=claude-sonnet-4-5
OPENAI_API_KEY=your_key_here
CLAUDE_API_KEY=your_key_here
```

## Supabase setup

Run the SQL in `src/ikenga/db/schema.sql` in your Supabase SQL editor before launch. The schema includes:

- `engines`
- `engine_versions`
- `engine_prompts`
- `engine_deployments`
- `engine_audit_logs`
- `waitlist_signups`

The public launch flow depends on `waitlist_signups`.

## Protected admin routes

All routes under `/api/engines/*` and `/api/ikenga/generate` are private and require the admin token:

```bash
Authorization: Bearer <IKENGA_ADMIN_API_TOKEN>
```

You can also send:

```bash
x-ikenga-admin-token: <IKENGA_ADMIN_API_TOKEN>
```

Without that token, the routes return `401`. If the token is not configured on the server, they return `503`.

## IKENGA generation API

`POST /api/ikenga/generate` returns structured JSON with:

- 7-day content calendar
- 14 social posts
- 7 short-form video scripts
- 7 carousel outlines
- 7 email drafts
- 3 ad variations
- 1 brand voice summary

JSON request example:

```bash
curl -X POST http://localhost:3000/api/ikenga/generate \
  -H "Authorization: Bearer $IKENGA_ADMIN_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "brand": "Ikenga AI",
    "niche": "AI content automation",
    "goals": "book strategy calls and grow the email list",
    "audience": "founders and lean marketing teams",
    "offer": "done-with-you content operating system",
    "tone": "confident, sharp, practical",
    "contentPillars": ["authority", "proof", "conversion"],
    "callsToAction": ["book a strategy call", "join the waitlist"]
  }'
```

Multipart request example with brand files:

```bash
curl -X POST http://localhost:3000/api/ikenga/generate \
  -H "Authorization: Bearer $IKENGA_ADMIN_API_TOKEN" \
  -F "brand=Ikenga AI" \
  -F "niche=AI content automation" \
  -F "goals=book strategy calls and grow the email list" \
  -F "audience=founders and lean marketing teams" \
  -F "offer=done-with-you content operating system" \
  -F "contentPillars=authority,proof,conversion" \
  -F "callsToAction=book a strategy call,join the waitlist" \
  -F "files=@./brand-brief.pdf"
```

## Operational routes

- `/api/health` returns launch-readiness checks for site URL, Supabase, the admin token, and Anthropic credentials.
- `/robots.txt` and `/sitemap.xml` are generated by the App Router metadata conventions.


## Deploy

Recommended target: Vercel.

1. Import the repository into Vercel.
2. Add the environment variables from `.env.example`.
3. Run the SQL from `src/ikenga/db/schema.sql` in Supabase.
4. Deploy.
5. Confirm `https://your-domain.com/api/health` returns `200`.
